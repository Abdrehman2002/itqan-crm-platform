# Bug Report: /api/v1/sales/templates returns raw pg result object instead of rows

## Severity
High

## Environment
- API: http://129.121.115.99:3000 (production)
- Codebase: `C:\Users\Thinkbook 16 G6\Downloads\itqan\itqan`
- Source: `packages/api/src/routes/sales/invoice-templates.ts` line 65
- Tested: 2026-06-19 21:44 UTC

## Steps to Reproduce
1. Login as tenant_admin and obtain JWT.
2. GET `/api/v1/sales/templates` with `Authorization: Bearer <token>`.

## Expected Behavior
Per contract used by every other list endpoint:
```json
{"success": true, "data": [ ...rows... ]}
```

## Actual Behavior
HTTP 200 but `data` is the entire pg `QueryResult` object, not the rows:
```json
{
  "success": true,
  "data": {
    "command": "SELECT",
    "rowCount": 0,
    "oid": null,
    "rows": [],
    "fields": [{"name":"id","tableID":19176,...}, ... ]
  }
}
```

Frontend code that does `data.map(...)` or `data.length` will break.

## Evidence
Source `packages/api/src/routes/sales/invoice-templates.ts`:

```ts
// Line 57-65
const rows = await db.withTenant(tenantId, (client) =>
  client.query(
    `SELECT id, name, type, ... FROM invoice_templates WHERE tenant_id=$1 ORDER BY ...`,
    [tenantId]
  )
);
return reply.send({ success: true, data: rows });
```

Bug: `client.query()` returns a `QueryResult`, not an array of rows. The variable is misnamed `rows` but holds the full result. Compare to line 73 of the same file which correctly destructures: `const [row] = await db.withTenant(...)`.

Fix: change line 57 to `const { rows } = await db.withTenant(tenantId, (client) => client.query(...))` (destructure) — or change line 65 to `data: rows.rows`.
