# Bug Report: /api/v1/opportunities returns HTTP 500 — column `c.name` does not exist on contacts

## Severity
Critical

## Environment
- API: http://129.121.115.99:3000 (production)
- Codebase: `C:\Users\Thinkbook 16 G6\Downloads\itqan\itqan`
- Source: `packages/api/src/routes/opportunities.ts` (lines 87, 90, 113, 117)
- Tested: 2026-06-19 21:45 UTC

## Steps to Reproduce
1. POST `/auth/login` with `{"email":"abdurrehman1711@gmail.com","password":"demo123","tenantSlug":"vextria"}` to obtain a tenant_admin JWT.
2. GET `/api/v1/opportunities` with `Authorization: Bearer <token>` (also tried `?page=1&pageSize=10`).
3. Also reproduced on POST `/api/v1/opportunities` with valid body `{"title":"QA Test Opp","value":1000,"currency":"USD"}` (the LIST after insert is what fails).

## Expected Behavior
HTTP 200 returning the list of sales opportunities for the tenant (the table exists, the data exists — there is already 1 row in `sales_opportunities` for tenant `bea2915f-...`).

## Actual Behavior
HTTP 500 with body `{"success":false,"error":{"code":"INTERNAL_ERROR","message":"An unexpected error occurred"}}`. The entire endpoint (LIST and GET-by-id) is unusable. Sales pipeline UI cannot render.

## Evidence
Direct SQL reproduction of the LIST query the route runs:

```sql
SELECT o.*, u.name AS assignee_name, c.name AS contact_name
FROM sales_opportunities o
LEFT JOIN users u ON u.id = o.assignee_id
LEFT JOIN contacts c ON c.id = o.contact_id
WHERE o.tenant_id = 'bea2915f-ac96-4d02-b61e-bc731ce66e04'
LIMIT 1;
-- ERROR: 42703: column c.name does not exist
-- HINT: Perhaps you meant to reference the column "u.name".
```

The `contacts` table has `first_name` and `last_name` only, no `name` column:
```sql
SELECT column_name FROM information_schema.columns
WHERE table_schema='public' AND table_name='contacts' AND column_name IN ('name','first_name','last_name');
-- [{"column_name":"first_name"},{"column_name":"last_name"}]
```

The same broken reference appears in:
- `opportunities.ts:87` (LIST query) → `c.name AS contact_name`
- `opportunities.ts:90` (LIST query) → ditto
- `opportunities.ts:113`, `117` (GET-by-id query) → `c.name AS contact_name, c.email AS contact_email`

Fix: replace `c.name` with `COALESCE(c.first_name || ' ' || c.last_name, c.first_name, c.last_name)` (or whatever the project's canonical "display name" expression is for contacts — see how the contacts list endpoint formats names).
