## Issues Found & Fixed

### BUG-2026-06-23-A ‚Äî CRITICAL ‚Äî tenant_admin couldn't load dashboard
- **Severity:** Critical (user-blocking)
- **Reproduction:** Login as `abdurrehman1711@gmail.com` ‚Üí /dashboard ‚Üí ops-dashboard widget tries `/api/v1/analytics/ops-dashboard` ‚Üí 403 ADMIN_NO_OPERATIONS
- **Root cause:** Munir's e69bd61 added a global block: any tenant_admin request to `/api/v1/{tickets,contacts,deals,analytics,...}` was rejected regardless of method. The block was too aggressive ‚Äî tenant_admin can't oversee what they can't see.
- **Fix:** server.ts ‚Äî only block POST/PATCH/DELETE for those prefixes. GET passes through. Removed `/api/v1/analytics` from the block list entirely (dashboards are read-only).
- **Commit:** `834f21d`
- **Re-test:** PASS ‚Äî tenant_admin /dashboard loads, GET /api/v1/tickets returns 200.

### BUG-2026-06-23-B ‚Äî HIGH ‚Äî `/super-admin/reports/audit` returned 500
- **Severity:** High
- **Reproduction:** `curl -H "Authorization: Bearer $SUPER_TOK" /super-admin/reports/audit`
- **Actual:** `500 Internal Server Error` ‚Äî column `tal.entity_type` does not exist
- **Root cause:** Munir's reports SQL queried `ticket_audit_log.entity_type` + `entity_id`, but the table (created earlier) only has `ticket_id`, `action`, `actor_*`, `old/new_value`. No generic entity columns.
- **Fix:** Rewrote SQL to synthesize `entity_type='ticket'::text AS entity_type, ticket_id AS entity_id`. Dropped the `?entity=` filter (only one entity type logged anyway).
- **Commit:** `834f21d`
- **Re-test:** PASS ‚Äî endpoint returns 200, real audit rows.

### BUG-2026-06-23-C ‚Äî HIGH ‚Äî `/super-admin/reports/invoices` returned 500
- **Severity:** High
- **Reproduction:** `curl -H "Authorization: Bearer $SUPER_TOK" /super-admin/reports/invoices`
- **Actual:** `500 Internal Server Error` ‚Äî column `pi.amount_paid` does not exist
- **Root cause:** Migration `023_platform_invoices.sql` created `amount` but not `amount_paid`. Munir's `/reports/invoices` SQL references both.
- **Fix:** Direct DB migration: `ALTER TABLE platform_invoices ADD COLUMN amount_paid NUMERIC(12,2) NOT NULL DEFAULT 0;` + backfilled from `SUM(platform_payments.amount)` per invoice.
- **Commit:** SQL applied via Supabase; column now persists.
- **Re-test:** PASS ‚Äî endpoint returns 200, computed `payment_status` and `due_status` rows.

### BUG-2026-06-23-D ‚Äî MEDIUM ‚Äî `/super-admin/users/:id/reset-password` 404 after merge
- **Severity:** Medium
- **Reproduction:** Login as super_admin ‚Üí POST to that path ‚Üí 404 "Route not found"
- **Root cause:** Wholesale overlay of Munir's super-admin.ts in commit `5892196` wiped my generic reset-password endpoint. Munir's `/tenants/:id/reset-admin-password` only resets the first tenant_admin of a tenant ‚Äî can't reset sub-admins, managers, agents, or non-first tenant_admins.
- **Fix:** Re-added endpoint. Also wired the `super_admin_password_log` audit table (from migration 025) so all super-admin resets are logged.
- **Commit:** `e0e7d6e`
- **Re-test:** Pending VPS deploy. Code grep confirms endpoint registered.
